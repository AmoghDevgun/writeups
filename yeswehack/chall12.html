CTF Challenge Writeup

Description

Small client-side challenge: a closure builds a Secret class that stores a flag on this.flag
It gets validates by dispatching CustomEvent('validate', { detail: text }).
The listener compares e.detail === this.flag.

Code 

                    
<html>
    <head></head>
    <body>
        <p id="result"></p>
        <script>
            const validate = (function(){
                const SecretEvent = CustomEvent;
                class Secret extends EventTarget {
                    constructor() {
                        super();
                        this.flag = `FLAG{${parseInt(Math.random() * Number.MAX_SAFE_INTEGER).toString(36)}}`;
                        this.addEventListener('validate', e => {
                            if (e.detail === this.flag){
                                result.innerHTML = 'Good! But can you leak the flag?';
                            }
                            else {
                                result.innerHTML = 'Wrong password!';
                            }
                        })
                    }
                    validate(text) {
                        this.dispatchEvent(new SecretEvent('validate', {detail: text}))
                    }
                }
                return text => {new Secret().validate(text)};
            })();
        </script>
        <script>
            EventTarget = null;
            validate("hello");
        </script>
    </body>
</html>

What’s happening

The page reads e.detail while the secret is on event.target. If we can run code at the moment e.detail is read, we can access this.target.flag and leak the flag.

The exploit (concept)

Replace CustomEvent.prototype.detail with a getter. When the page reads e.detail, the getter executes with this bound to the event so this.target is the Secret instance and this.target.flag is the flag.

PoC


new CustomEvent('')['__proto__'].__defineGetter__('detail', function(){
  alert(this.target.flag);
});

TL;DR

Mutate CustomEvent.prototype.detail → getter runs when e.detail is read → this.target.flag leaks.
