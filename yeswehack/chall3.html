CTF Challenge Writeups

Description

This challenge involves two user-controlled fields injected directly into a <script> tag: 

                    
<script>
const age = 
const name = ""
if (age > 18 && name.length)
{
    document.write(Welcome to my adult website ${name} 8=D) 
} 
else 
{ 
    document.write(Go away) 
} 
</script>

Constraints

  • age: Maximum 2 characters, no blacklist.
  • name: Blacklist forbids <, >, |, ', ".

Goal

Execute a XSS alert(window.name)

Exploitation Steps

  • Put a single backslash in the age field: \
  • This causes the line to be continued into the next line when the server inserts it into the script source.
  • Craft name to start with a semicolon (to end whatever expression is open), add the payload, and finish with // to comment out leftover characters: ;alert(window.name)//
  • When combined, the injected text becomes a new executable statement, and the alert runs.

Proof of Concept (PoC)

Submit the following

  • age: \
  • name: ;alert(window.name)//
Upon submission, you should see the alert popup.

Why It Works

The backslash at the end of the age line escapes the newline in the generated script, so the name line no longer behaves as a simple assignment. Starting name with ; creates a fresh statement, and // comments out the rest, keeping the script syntactically valid. No <, >, or quotes are needed, bypassing the blacklist.
